3) LangGraph StateGraph 구조

노드 (12개)

securitycontext_loadplanner memoryslotstools responderjudgeretry_prep finalize_blockedfinalize_memoryfinalize_answer

조건부 분기 (conditional edges) 3곳

분기 노드조건이동
security민감정보/과다입력 감지차단 → finalize_blocked / 통과 → context_load
memory메모리 명령 처리 완료 여부처리됨 → finalize_memory / 통과 → slots
judge재시도 필요 & 상한(1회) 이내retry → retry_prep → tools / 그 외 → finalize_answer

반복 루프

judge → retry_prep → tools (최대 1회, 무한 루프 방지)

4) 단계별 내부 동작 & LLM 개입

단계(노드)역할LLM
security민감정보(주민번호/카드/전화/여권), 입력 길이 검사규칙 기반 (미들웨어)
context_load단기 대화창 + 장기 프로필/요약/최근대화 로드없음
planner의도 분석, 메모리명령 판단, 툴 계획, 슬롯 추출LLM 선택 모델
memory장기기억 ON/OFF·수정·삭제 FSM (동의/취소 확인)규칙 기반 (미들웨어)
slots지역/나이/소득/결혼/분야 표준값 정규화없음 (맵핑 테이블)
tools정책RAG·컨텐츠RAG·웹검색·파일파싱 오케스트레이션없음 (검색·평가)
responder툴 결과 + 맥락으로 최종 답변 초안 작성LLM 선택 모델
judge질문-근거-답변 정합성 평가, 재시도/되물음 결정LLM 선택 모델
Planner / Responder / Judge 세 곳에서 사용자가 상단에서 고른 모델이 그대로 실행됩니다.

입력 길이 제한 과도한 길이 요청 차단

6) Tool / RAG / Memory

Tool (4종, 오케스트레이션으로 복수 동시 실행 가능)

rag_policy_search rag_content_search web_search file_parse

RAG

  • 정책 / 컨텐츠 분리 벡터스토어 운영
  • 정책 검색: 하드조건(지역·나이·결혼·소득·분야) 불일치 감점 + 의미 유사도 가점
  • 1차 추림 후 Judge LLM이 최종 정합성 평가

Memory

  • 단기: 세션 대화창 (in-memory)
  • 장기: 유저코드 기반 SQLite (프로필 + 요약 + 최근대화)

7) 상세 사용 가이드

기본 사용 순서

1
서버 실행: python -m agent3.run_server (포트 8002)
2
브라우저 접속: http://localhost:8002
3
상단에서 모델 선택 → 질문 입력 → 전송
4
필요 시 📎로 파일 첨부 후 파일 기반 질문
5
우측 패널에서 계획·슬롯·툴·메모리 상태 실시간 확인

장기기억 사용법

  1. 상단 유저코드 입력 (예: user_kim_01)
  2. "장기기억 켜줘" 또는 "내 프로필 저장할게"로 요청
  3. 시스템이 확인을 물으면 "동의" 또는 "확인"으로 확정 (취소하려면 "취소")
  4. 나이·지역·소득·결혼여부·관심분야를 자연어로 알려 프로필 갱신
  5. "내 저장 프로필 보여줘"로 조회, "장기기억 꺼줘"로 비활성화

유저코드 규칙

  • 유저코드는 장기기억 식별자입니다. 같은 코드 = 같은 프로필·이력 연결
  • 다른 코드를 쓰면 완전히 독립된 메모리 공간
  • 유저코드를 비우면 장기기억 없이 단기 대화만 유지

프로필 저장 방식

  • 장기기억이 ON일 때만 프로필/대화가 SQLite에 저장됩니다
  • 슬롯(지역·나이·소득·결혼·주거·분야) 단위로 표준화되어 보관
  • 다음 대화에서 자동으로 컨텍스트에 주입되어 맞춤 추천에 반영

모델 선택 가이드

  • 정확한 구조화·추론 중심: 고성능 모델
  • 빠른 응답·저비용 중심: 경량 모델
  • 선택 모델은 Planner·Responder·Judge 3단계에 반영

질문 방식 & 추천

  • 맞춤 추천: "서울 거주, 27세, 미혼, 중위소득 100%, 취업 분야 정책 추천"
  • 컨텐츠 탐색: "청년 대상 모집 공고나 최신 소식 알려줘"
  • 최신성 강화: "최근 마감 임박 정책 위주로 알려줘"
  • 파일 기반: 파일 첨부 후 "이 파일 조건에 맞는 정책만 추천해줘"
