청년정책 Agent 동작 보고서

실행 흐름, LLM 개입 지점, 미들웨어, Tool 오케스트레이션, 사용법 중심 문서
이 페이지 하나로 입력부터 최종 응답까지 내부 동작과 사용자 사용법을 모두 파악할 수 있습니다.

5) 상세 사용 가이드

기본 사용 순서

  1. 서버 실행: python -m agent2.run_server
  2. 브라우저 접속: http://localhost:8001
  3. 상단에서 모델 선택 후 질문 입력
  4. 필요 시 파일 첨부 후 파일 기반 질문 수행
  5. 우측 패널에서 계획, 슬롯, Tool 실행, 메모리 상태를 확인

장기기억 사용법

  1. 유저코드를 입력합니다. (예: user_kim_01)
  2. "장기기억 켜줘" 또는 "내 프로필 저장할게"처럼 명령합니다.
  3. 시스템이 동의/확인을 요청하면 "동의" 또는 "확인"으로 확정합니다.
  4. 이후 나이, 지역, 소득, 결혼여부, 관심분야 등을 자연어로 알려 프로필을 업데이트합니다.
  5. "내 저장 프로필 보여줘"로 저장 상태를 조회합니다.

유저코드 사용 규칙

  • 유저코드는 장기기억 식별자입니다. 같은 유저코드를 쓰면 같은 프로필/이력이 연결됩니다.
  • 다른 유저코드를 쓰면 독립된 메모리 공간으로 동작합니다.
  • 테스트와 실사용을 분리하려면 유저코드를 구분해서 사용하세요.

모델 선택 가이드

  • 정확한 구조화/추론 중심: 고성능 모델 선택
  • 빠른 응답/저비용 중심: 경량 모델 선택
  • 선택한 모델은 Planner, Responder, Judge 단계에 반영되어 실행됩니다.

질문 방식과 추천 방향

  • 좋은 질문 예시: "서울 거주, 27세, 미혼, 중위소득 100%, 취업 분야 정책 추천"
  • 컨텐츠 중심 예시: "청년 대상 모집 공고나 최신 소식 알려줘"
  • 최신성 강화 예시: "최근 마감 임박 정책 위주로 알려줘"
  • 파일 기반 예시: 파일 업로드 후 "이 파일 조건에 맞는 정책만 추천해줘"
보고서 페이지 URL: /static/assignment_evaluation_report.html

6) 내부 동작 상세

Tool rag_policy_search, rag_content_search, web_search, file_parse
RAG 정책/컨텐츠 분리 벡터스토어 + 재랭킹
Memory 단기(SessionStore) + 장기(SQLite LongTermStore)
Judge Loop 최대 1회 재시도로 무한 루프 방지
Middleware 성격 보안 필터/입력 길이 제한/예외 반환/메모리 확인 절차

7) 제출 시 강조 포인트

  • 입력부터 응답까지 LLM 개입 지점(Planner/Responder/Judge)을 단계별로 설명하세요.
  • 미들웨어가 단순 로깅이 아닌 안전장치(민감정보/확인절차)로 작동함을 보여주세요.
  • Tool 오케스트레이션으로 단일/복수 Tool 실행이 가능한 흐름을 시연하세요.
  • 유저코드 기반 장기기억 재사용 시나리오를 라이브 데모로 보여주세요.
